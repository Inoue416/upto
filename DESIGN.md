# Upto Design Guidelines

作成日: 2026-06-13

Upto は日本語ITニュースを短時間で読むためのプロダクト。UIは速く読めること、次の記事へ迷わず進めること、スマートフォンで片手操作できることを優先する。

## 基本方針

- 最初の画面はニュースフィードにする。マーケティングページや説明用ヒーローにしない。
- 情報密度は高めに保つ。装飾よりもタイトル、要約、出典、公開時刻、タグを読みやすくする。
- モバイルは縦スワイプ、デスクトップはホイールやトラックパッドで1記事ずつ進む体験を維持する。
- 新しい画面を追加するときは `apps/web/src/app/**/page.stories.tsx` を同じディレクトリに置き、主要状態をStorybookで確認できるようにする。

## 色

現在のテーマは `apps/web/src/app/globals.css` のCSS変数を正とする。

| 用途           | 変数              | 値        |
| -------------- | ----------------- | --------- |
| 背景           | `--background`    | `#faf8f1` |
| 本文           | `--foreground`    | `#1d211c` |
| 補助テキスト   | `--muted`         | `#62695f` |
| 面             | `--surface`       | `#ffffff` |
| 弱い面         | `--surface-muted` | `#fffdf7` |
| 境界線         | `--border`        | `#e6e0d2` |
| 強い境界線     | `--border-strong` | `#cfc6b6` |
| アクセント     | `--accent`        | `#2f735d` |
| 弱いアクセント | `--accent-soft`   | `#e3f0e8` |

新しい色を追加する前に、既存変数で表現できないか確認する。紫グラデーション、濃いスレート一色、過度な装飾用ブロブは使わない。

## レイアウト

- フィード本体は `h-dvh` と CSS scroll snap を基準にする。
- コンテンツ幅は現行の `max-w-3xl` を基準にし、長文が横に伸びすぎないようにする。
- カードやバッジの角丸は `rounded-md` から `rounded-lg` 程度に抑える。
- セクション全体を何重にもカード化しない。繰り返し項目、状態表示、短い注釈に限定する。
- モバイルでは本文の折り返しと `line-clamp` を優先し、要素同士が重ならない高さにする。

## タイポグラフィ

- フォントは現行の system sans stack を使う。
- 見出しはニュースタイトルを主役にする。ヒーロー級の巨大文字は使わない。
- 文字間隔は通常値を保ち、負の letter spacing は使わない。
- 日本語コピーは短く具体的にする。ユーザー向けの表示文言は原則日本語にする。

## インタラクション

- キーボード操作は Up が前の記事、Down が次の記事、Space が次の記事、Shift+Space が前の記事。
- フォーカス可能要素には見える focus style を残す。
- 外部記事リンクは新規タブで開き、`rel="noreferrer"` を付ける。
- ローディング、空状態、エラー状態は画面ごとに用意する。

## Storybook

- ページを増やしたら、同じディレクトリに `page.stories.tsx` を追加する。
- 小さい内部コンポーネントの stories は、ユーザーから指摘があるまで必須にしない。
- Storybookでは通常状態、空状態、エラーや読み込みなどの主要状態を優先して表現する。
